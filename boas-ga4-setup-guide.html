중요 안내

먼저 Google Cloud 프로젝트를 생성하고 서비스 계정을 만든 후, 해당 이메일을 Google Search Console과 Analytics에 추가해야 API가 정상 작동합니다.

서비스 계정 이메일 (복사하여 사용)

TODO_서비스계정@TODO_프로젝트.iam.gserviceaccount.com

아래 Section 0에서 Google Cloud 프로젝트 생성 후 이 이메일을 업데이트하세요.

Google Cloud 프로젝트 설정

서비스 계정을 생성하기 위한 기본 환경 구성

1

Google Cloud Console 접속

Cloud Console 열기

💡 Google 계정으로 로그인하여 시작하세요.

2

새 프로젝트 생성 또는 기존 프로젝트 선택

화면 상단의 프로젝트 선택 드롭다운 클릭:

프로젝트 선택 → 새 프로젝트

프로젝트 정보 입력:

boas-analytics

또는 기존 프로젝트를 선택하세요.

3

서비스 계정 생성

☰ 메뉴 → IAM & Admin → Service Accounts → + CREATE SERVICE ACCOUNT

서비스 계정 정보 입력:

boas-analytics

자동 생성됩니다 (예: boas-analytics@프로젝트ID.iam.gserviceaccount.com)

Step 2 & 3 (Role/Users): Skip 가능 → Done 클릭

4

JSON 키 생성 및 다운로드

생성된 서비스 계정 클릭 → KEYS 탭 → ADD KEY → Create new key

키 타입 선택

JSON ← 이 옵션 선택

⚠️ JSON 파일이 자동 다운로드됩니다. 안전한 곳에 보관하세요!

5

서비스 계정 이메일 확인 및 기록

Service Accounts 목록에서 이메일 확인:

boas-analytics@프로젝트ID.iam.gserviceaccount.com

이 이메일을 복사하여 페이지 상단의 "서비스 계정 이메일" 섹션과 아래 Section 2, 3에서 사용하세요.

✅ Google Cloud 프로젝트 설정 완료! 이제 API 활성화로 넘어가세요.

G-TCC437319B

설정 완료 체크

모든 설정이 완료되면 API가 정상 작동합니다

Search Console

  • ✓ 서비스 계정 이메일 추가됨
  • ✓ 전체(Full) 권한 부여됨

Analytics

  • ✓ 서비스 계정 이메일 추가됨
  • ✓ 뷰어 이상 권한 부여됨

환경변수

  • ✓ 4개 환경변수 설정됨
  • ✓ Vercel에도 동일하게 설정됨

💡 참고: 권한 설정 후 API가 작동하기까지 최대 10분 정도 소요될 수 있습니다.

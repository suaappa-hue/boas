⑤ Worker 처리 흐름 (참고) 

[Make.com] → POST /meta-lead (boas-notify Worker)
              ↓
              연락처 정규화 (+82 → 0, 비숫자 제거)
              ↓
              Airtable 블랙리스트 조회 (5분 Cache API 캐시)
              ↓
              ┌─ 매치 안됨 → 기존 흐름 (상태=신규 + 텔레그램 발송)
              └─ 매치   → 상태='스팸' 으로 Airtable 저장
                        + 차단횟수 +1, 마지막차단일시 업데이트
                        + 텔레그램 발송 SKIP
                        + 200 OK 반환

⑥ 추가 확인 필요 항목

  • 전화번호 정규화 규칙: 비숫자 제거 후 +82 prefix → 0 으로 치환. 예) +821077556700 → 01077556700, 010-7755-6700 → 01077556700. 정영태처럼 자릿수 이상한 번호도 그대로 정규화 후 매칭.
  • 홈페이지 폼 적용 여부: 메타 리드 외에 홈페이지 상담 폼(`/`)에도 블랙리스트 차단을 적용할지 — 적용 권장.
  • Airtable 블랙리스트 테이블 생성: 메타데이터 API로 자동 생성하기 까다로워서, 사용자가 Airtable UI에서 수동 생성하는 게 안전함. 필드 가이드 제공 예정.
